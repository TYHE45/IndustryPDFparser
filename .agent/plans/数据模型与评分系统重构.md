## 1. 模块目标

### 1A. 数据模型字段中文化
`src/models.py` 中 `DocumentData` 的 13 个属性中仍有 10 个英文（`metadata / raw_pages / sections / tables / numeric_parameters / rules / inspections / standards / blocks / profile`），只有 3 个中文（`页面列表 / 结构节点列表 / 产品列表`）。`DocumentProfile` 的 16 个属性中多数仍为英文（`doc_type / confidence / language / layout_mode / has_many_tables ...`），虽 `to_dict()` 产出中文键，但 Python 侧仍暴露英文属性。

目标：按 FP §6 的承诺，把内部属性也中文化，保留 `OCR / PDF / JSON / LLM / API / DPI / ID` 这类约定俗成英文缩写。废除已不再使用的 v2 语义分支（如有）。

### 1B. 评分系统重构
`src/reviewer.py`（1254 行）当前评分表达（行 147-158）虽然总量已经对齐 35/40/25，但组合方式是「4 个模块分（markdown/summary/tag/source）→ 3 个维度分（base/factual/consistency）用 min()+加权拼装」，难以解释每个扣分项对最终分的贡献。

同时文件内有大量重复函数定义（Python 只使用最后一份，前面的是死代码）：
- `_review_metadata_consistency` ×3（行 441 / 475 / 538；其中 475 是 mojibake 乱码版）
- `_review_table_criticality` ×2（行 520 / 550）
- `_problem_meta` ×2（行 717 / 905）
- `_is_suspicious_ocr_heading` ×2（行 1127 / 1147）
- `_looks_like_template_summary` ×3（行 1208 / 1219 / 1232）

目标：清除所有重复函数（保留最后一份）+ 重写评分主流程，把扣分项→维度的映射摆平，红线规则统一到 FP §10 的三条。

## 3. 1B — 评分系统重构

### 3.1 评分框架（严格对齐 FP §10）

```
总分 = 基础质量分 + 事实正确性分 + 一致性与可追溯性分
       (满分 35)     (满分 40)        (满分 25)

红线触发 → 总分上限 75（<85 → 强制不通过）

通过条件：总分 ≥ 85 且 红线列表为空 且 无 S 级阻塞问题
```

### 3.2 三大维度扣分细则

#### 基础质量分（35）— 章节完整性 / 表格提取 / markdown 结构清洁度
| 扣分项 | 扣分幅度 | 现有检测函数 |
|-------|---------|------------|
| `章节列表` 为空 | −10 | `_review_markdown` |
| markdown 标题 ≤2 或正文行 ≤3 | −8 | `_review_markdown` |
| 已抽到表格但 markdown 无表格视图 | −5 | `_review_markdown` |
| markdown 残留系统生成的「第N页表K」标题 | −4 | `_review_markdown` |
| 伪标题数量 > 5 | −3 | `_review_markdown.PSEUDO_HEADINGS` |

#### 事实正确性分（40）— summary/tags 与原文一致性 + 参数值准确性
| 扣分项 | 扣分幅度 | 现有检测函数 |
|-------|---------|------------|
| LLM 自述无内容模板句 | −10 | `_review_summary_llm_stub` |
| 已抽参数但 summary 无数值条目 | −8 | `_review_summary_facts` |
| metadata 与内容标准号冲突 | −10 | `_review_metadata_consistency`（最后一份） |
| 表格驱动型文档但未抽到核心表格 | −8 | `_review_table_criticality`（最后一份） |
| 参数/标签存在噪音句子 | −5 | `_review_tags` |
| 关键标签缺失（标准/参数/产品型号） | −4 | `_review_tags` |

#### 一致性与可追溯性分（25）— 跨文件引用一致 + 原文可溯源
| 扣分项 | 扣分幅度 | 现有检测函数 |
|-------|---------|------------|
| 参数来源异常（页码/章节缺失） | −8 | `_review_sources` |
| 规则来源异常 | −6 | `_review_sources` |
| OCR 标题噪音 ≥5 条 | −6 | `_review_ocr_quality` |
| OCR 标题噪音 2–4 条 | −3 | `_review_ocr_quality` |
| 章节引用不一致 | −5 | `_review_sources` |

每个维度最低 0 分。

### 3.3 红线规则（FP §10）

| 红线名称 | 触发条件 | 分数上限 |
|---------|---------|---------|
| 正文主链缺失 | `章节列表` 空 且 markdown 骨架不够 | 74 |
| 表格存在但参数未建立 | `表格列表` 非空 且 `数值参数列表` 为空 | 74 |
| 文本层不足需要OCR | `文档画像.是否需要OCR=True` 且 未成功注入 OCR 到正文主链 | 74 |

**规则**：任一红线触发 → `总分 = min(总分, 74)` → 强制不通过。上限值 74 而非 75，避免浮点边界上打出 75.0 的歧义；与 FP §10「上限降至 75 以下」一致。

### 3.4 `review.json` 字段结构

严格对齐 `.agent/skills/fp-review-output/SKILL.md` 的 JSON 模板，并兼容现有 `review_rounds.json` 字段：

```json
{
  "轮次": 1,
  "总分": 0.0,
  "是否通过": false,
  "基础质量分": 0.0,
  "事实正确性分": 0.0,
  "一致性与可追溯性分": 0.0,
  "红线触发": false,
  "红线列表": [
    {"红线名称": "...", "分数上限": 74, "触发原因": "..."}
  ],
  "问题清单": [
    {"等级": "红线|S|A|B", "模块": "parser|md_builder|summarizer|tagger|fixer", "描述": "...", "建议修正动作": "..."}
  ],
  "问题统计": {"S级问题数": 0, "A级问题数": 0, "B级问题数": 0},
  "分项评分": {
    "基础质量各扣分点": [...],
    "事实正确性各扣分点": [...],
    "一致性各扣分点": [...]
  },
  "文档类型": "standard"

```

变动：
- 删除旧 key `最终总评 / 最终通过`（与 `总分 / 是否通过` 重复）
- 删除旧 key `markdown检查 / summary结构检查 / summary事实检查 / tags检查 / OCR专项检查 / 一致性检查 / 来源锚点检查 / 类型专项检查` — 这些是中间调试输出，合并为 `分项评分` 的三层汇总

- `关键问题 / 必须修复的问题` 合并为 `问题清单` 中 `等级=红线|S` 的子集即可

## 4. 数据流图

```
Web UI / CLI
    │
    ▼
pipeline.run_pipeline(pdf_path, config)
    │
    ▼
parser.parse(pdf_path) ──► DocumentData（中文属性）
    │  ├─ 文件元数据
    │  ├─ 原始页面列表 / 页面列表（含 OCR 字段）
    │  ├─ 章节列表 / 结构节点列表
    │  ├─ 表格列表
    │  ├─ 数值参数列表 / 规则列表 / 检验列表 / 引用标准列表
    │  ├─ 内容块列表 / 产品列表
    │  └─ 文档画像（中文属性）
    │
    ▼
md_builder / summarizer / tagger（只读 DocumentData）
    │
    ▼
reviewer.review_outputs(document, markdown, summary, tags)
    │  ┌─ _review_markdown           ─► 基础质量扣分点
    │  ├─ _review_summary_structure  ─┐
    │  ├─ _review_summary_facts      ─┼─► 事实正确性扣分点
    │  ├─ _review_summary_llm_stub   ─┤
    │  ├─ _review_metadata_consistency ┘
    │  ├─ _review_table_criticality
    │  ├─ _review_tags
    │  ├─ _review_sources            ─► 一致性与可追溯性扣分点
    │  ├─ _review_ocr_quality
    │  ├─ _review_skeleton
    │  └─ _detect_redlines           ─► 红线列表
    │
    ▼
review.json（字段按 3.4 结构） → review_rounds.json 追加
    │
    ▼
通过 → exporter（12 份输出）；未过且 N<3 → fixer 按路由修正 → 回 reviewer
```
